# دیپلوی بات روی Railway.app

## مرحله ۱ - ساخت پروژه روی Railway

1. وارد [railway.app](https://railway.app) شو و با گیت‌هاب لاگین کن.
2. کد رو (فایل‌های `bot.py`, `requirements.txt`, `Procfile`, `.gitignore`) توی یک ریپازیتوری گیت‌هاب جدید آپلود کن.
   **فایل `.env` رو هرگز آپلود نکن** — توی `.gitignore` گذاشتمش که خودکار نادیده گرفته بشه.
3. توی Railway: `New Project` → `Deploy from GitHub repo` → ریپازیتوری خودت رو انتخاب کن.

## مرحله ۲ - ست‌کردن متغیرهای محیطی (Environment Variables)

روی Railway فایل `.env` آپلود نمی‌شه، بلکه متغیرها رو مستقیم توی پنل سایت وارد می‌کنی:

1. توی پروژه‌ت روی سرویس کلیک کن → تب **Variables**
2. این دو تا رو اضافه کن:
   - `BOT_TOKEN` = `8403952516:AAEOLnuS7NL7dIyuFjFJYwe-4-C0EU4RBpM`
   - `ADMIN_ID` = `7663665401`
3. سیو کن — Railway خودش دوباره دیپلوی می‌کنه.

## مرحله ۳ - تنظیم نوع سرویس (Worker، نه Web)

این بات یک پروسه‌ی پس‌زمینه‌ست (polling)، نه یک سایت. Railway از روی فایل `Procfile` می‌فهمه که باید
دستور `worker: python bot.py` رو اجرا کنه. اگه پیغامی درباره‌ی "no exposed port" دیدی نگران نشو،
چون این بات نیازی به پورت نداره.

اگه Railway خودش `Start Command` رو خالی گذاشته، توی تب **Settings** سرویس، قسمت **Deploy** بخش
`Custom Start Command` رو روی این بذار:
```
python bot.py
```

## مرحله ۴ - بررسی لاگ‌ها

توی تب **Deployments** → روی آخرین دیپلوی کلیک کن → **View Logs**.
اگه خط `✅ Bot started successfully` رو دیدی یعنی بات بالا اومده. برو تو تلگرام به بات `/start` بزن.

## ⚠️ نکته‌ی مهم درباره‌ی ذخیره‌ی داده روی Railway

این بات اطلاعات کاربرها، پوزیشن‌های فعال و تاریخچه‌ی سیگنال‌ها رو توی دو فایل JSON کنار خودش
(`users_data.json` و `users_list.json`) ذخیره می‌کنه. روی Railway به‌صورت پیش‌فرض فضای ذخیره‌سازی
**ناپایدار (ephemeral)** هست؛ یعنی هر بار که دوباره دیپلوی کنی یا سرویس ری‌استارت بشه، این فایل‌ها
از بین می‌رن و کاربرها/پوزیشن‌های فعال پاک می‌شن.

دو راه‌حل داری:
1. **استفاده از Volume در Railway** (در پلن‌های پولی موجوده): توی تب Settings سرویس، یک Volume بساز و
   مسیرش رو روی پوشه‌ی پروژه ست کن تا فایل‌های JSON روی دیسک دائمی ذخیره بشن.
2. **استفاده از یک دیتابیس** (مثل Railway Postgres که رایگانه): اگه بخوای داده‌ها همیشه پایدار بمونن،
   بهتره کد رو طوری تغییر بدیم که به‌جای فایل JSON از یک دیتابیس کوچیک استفاده کنه. اگه خواستی این کار
   رو هم برات انجام بدم بگو.

برای تست و استفاده‌ی شخصی/کم‌حجم همین حالت فایل JSON کفایت می‌کنه، فقط بدون که با هر ری‌دیپلوی،
لیست کاربرها و تاریخچه‌ی سیگنال‌ها از نو شروع میشه.

## آپدیت کد در آینده

کافیه تغییرات رو به همون ریپازیتوری گیت‌هاب پوش کنی؛ Railway به‌صورت خودکار نسخه‌ی جدید رو
دیپلوی می‌کنه (Auto Deploy از تب Settings فعاله به‌صورت پیش‌فرض).

## یادآوری امنیتی

توکنی که الان توی `BOT_TOKEN` گذاشتی همون توکن قبلیه که یه بار توی کد دیده شده بود. توصیه می‌کنم
بعد از اینکه دیپلوی رو تست کردی و مطمئن شدی کار می‌کنه، بری پیش [@BotFather](https://t.me/BotFather)،
از منوی بات خودت `API Token` → `Revoke current token` بزنی، توکن تازه بگیری و فقط همونو توی
Variables پنل Railway جایگزین کنی.
